# タッチタイピング練習計画

## ポジション対応表

### アルファベット対応表
ホームポジションに指を置いた状態で始める

各指番号を以下の通り振る

1. 人差し指を体の内側に伸ばす: 0
2. 人差し指: 1
3. 中指: 2
4. 薬指: 3
5. 小指: 4
6. 小指を体の外側に伸ばす: 5

また、キーボードの各段については以下の通り番号を振る

1. 一番下の段: 0
    - ホームポジションから各指を折って使う
2. ホームポジション: 1
3. ホームポジションの上の段: 2
    - ホームポジションから各指を伸ばして使う
4. 一番上の段: 3

この組み合わせによって、アルファベットを以下の通り表現する

アルファベット | 左右,段,指
:--:           | :--:
a              | l14
b              | 親指
c              | l01
d              | l12
e              | l22
f              | l11
g              | l10
h              | r10
i              | r22
j              | r11
k              | r12
l              | r13
m              | r01
n              | r00
o              | r23
p              | r24
q              | l24
r              | l21
s              | l13
t              | l20
u              | l21
v              | l00
w              | l23
x              | l02
y              | r20
z              | l03

本表では、z, x, c, v を 左手の薬指, 中指, 人差し指, 人差し指（伸ばし）に対応させているが、一般的なタッチタイピングでは小指〜人差し指に対応させている

これは、左手の動きを右手と同一のものとするためにあえてこうしている

一般的なタッチタイピングでは、左手の最下段はやや内側に指を曲げて押すが、右手の最下段はやや外側に指を曲げて押すことになる

これを統一するために、最下段は両手ともやや外側に指を曲げることにしている

## ブラインドタッチ練習

### ファースト・ラーニング
ファースト・ラーニング段階では、意図的な行動が学習効率を向上させるため、一つのキーを押す際の行動を以下のような関数として定義する

```julia
# キー入力関数
## キー -> 行動
press_key(key) = begin
    set_home_position()   # 指をホームポジションに置く
    finger = key_map(key) # キーを [左右, 段, 指] に変換し、指のポジションを変更する
    press(finger)         # 指に押下処理を指令する
end
```

ここで目標とするのは、行動の律速処理となる `key_map`関数の高速化である

この処理は `キー -> [左右, 段, 指]` の変換であることから出力情報処理量は以下の式で表される

$$ \begin{array}{ll}
    output &= 2（左右）\times 4（段: 0〜3）\times 6（指: 0〜5） \\
           &= 48
\end{array} $$

経験的に、スキルに慣れていない段階では、出力情報1つあたりにかかる時間は、0.1秒 程度である

すなわち、タッチタイピング開始時点での1キーの押下にかかる時間は、およそ 4.8秒 程度であると計算できる

逆数をとると 0.2キー／秒 程度の速度感である

目標の第一段階では、このくらいの速度でキーを打ち間違えることなく処理できるようになることを目指す

ここでは**速度よりも正確性**の方が重要であるため、**1キーのポジションをしっかり意識して指を動かす**ことを覚える

### ファースト・ラーニング反復学習
具体的な反復学習の計画を立てる

ここでは学習効率のため、50秒の学習＋10秒の休憩を1セットとして反復練習を行う

前述のように 0.2キー／秒 の速度でタッチタイピングすると考えると、10キーの打鍵ごとに10秒の休憩を行うのが良いと考えられるため、以下のように計画する

```julia
# ファースト・ラーニング反復学習
first_learning() = begin
    # 10キー打鍵
    for key in "abcdefghij"
        press_key(key)
    end
    # 10秒休憩
    sleep(10)
    # 10キー打鍵
    for key in "klmnopqrst"
        press_key(key)
    end
    # 10秒休憩
    sleep(10)
    # 10キー打鍵
    for key in "uvwxyz1234"
        press_key(key)
    end
    # 10秒休憩
    sleep(10)
    # 10キー打鍵
    for key in "567890-^@;"
        press_key(key)
    end
    # 10秒休憩
    sleep(10)
    # 5キー打鍵
    for key in ":,./_"
        press_key(key)
    end
    # 10秒休憩
    sleep(10)
end
```

最初は時間を気にせず正確に打つことを心がけ、1キーを正確に打てる時間が 0.48秒（基準値の 1/10）程度になるまで反復練習を行う

また、1キーごとに必ずホームポジションに戻ってくることを忘れないようにする

### スロー・ラーニング
スロー・ラーニングは基本的に時間がかかるため、意識的学習と並行して行っていく

アルファベット一つ一つではなく、文章のようなつながりをタッチタイピングする訓練を行う

ファースト・ラーニングからスロー・ラーニングへの移行は明確に定まっていないため、タイプミスが頻発する時点では速さより正確性を意識して意図的行動を起こす（ファースト・ラーニング）ように心がける必要がある

脳内で十分な最適化が行われ、意識的行動が抑制され始めると自然にスロー・ラーニング状態に変化する

タイプミスがなくなってきたら、短期的反復学習よりも長期的反復学習を行うように計画を変更すれば良い

また、この段階では、フィードバック（KR）の頻度はある程度抑制されている方が効果的であるため、イメージトレーニングを取り入れることも重要である

例えば以下のような反復学習モデルが有効である

```julia
# スロー・ラーニング反復学習
slow_learning() = begin
    # e-typing等の演習問題
    typing()
    # 15分以上のエクササイズあるいは睡眠
    rand(0:1) === 0? sleep(60*60*6) : excercise(60*15)
    # e-typingの長文問題などをイメージトレーニング（実際にキーを押さずに演習する）
    typing(mode=VIRTUAL)
    # 以下繰り返し
    slow_learning()
end
```
